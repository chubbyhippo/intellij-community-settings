== Meow-style editing

Movement and editing mirror the Emacs `init.el` meow QWERTY layout: a
selection-first (Kakoune-like) model where you *select first, then act*. There
are no `d`/`c`/`y` operators (no `dd`, `cw`, `diw`, `yy`) — act on a whole line
with `x` (select line) then `s`/`d`/`c`/`y`.

[cols="1,3",options="header"]
|===
| Key | Action

| `h` `j` `k` `l` | Move, dropping the selection
| `H` `J` `K` `L` | Start / extend the selection
| `1`–`9` `0` `-` | Extend the selection by N lines (`0` = 10, `-` upward)
| `w` `W`         | Select word / WORD under the cursor (and arm grow mode)
| `e` `E`         | Bare `e` steps to the next word; after `w`, `e e` grows the mark right
| `b` `B`         | Bare `b` steps to the prev word; after `w`, `b b` grows the mark left
| `f` `t` (`F` `T`) | Select up to / till a character (backwards)
| `x`             | Select the line; tap to grow down (after `;`, grows up)
| `X`             | Go to line (`{count}X`)
| `o` `O`         | Expand / shrink the selection by syntax (meow-block)
| `;`             | Reverse the selection (swap its active end)
| `,` `.`         | Select the inner / bounds of a thing (then a char, e.g. `,(`)
| `i` `a`         | Insert at the start / end of the selection
| `I` `A`         | Open a line above / below
| `s`             | Kill / cut to the clipboard
| `d` `D`         | Delete forward / backward (black hole — keeps the clipboard)
| `c`             | Change
| `y` `p`         | Copy / paste
| `r`             | Replace the selection with the clipboard
| `m`             | Join lines
| `g`             | Cancel the selection
| `v`             | Visit — open Find; `n` / `N` then walk the matches
| `q`             | Quit the editor
| `u` `U`         | Undo / redo (the IDE's, shared with Emacs `C-/`)
| `'`             | Repeat the last change
| `S` `Q`         | Avy jumps: `S` to any 2 chars, `Q` to a line
|===

This follows `init.el`'s meow keys rather than vanilla Vim — `q`, `m`, `g`,
`v`, `,`/`.`, `'` and `u`/`U` all take their meow meanings above. Because the
*Emacs keymap* is used in IntelliJ, every Emacs editing chord is routed to the
IDE and works in all modes: `C-a`/`C-e` line ends, `C-f`/`C-b`/`C-n`/`C-p`
motion, `C-d` delete, `C-k` kill-line, `C-w`/`M-w` cut/copy, `C-y` yank,
`C-space` set mark, `C-g` cancel, `C-s`/`C-r` search, the `C-x` prefix, and
`M-x`. Only `C-[` stays with Vim, as the way back to normal mode. The leader is
`<Space>` (also `Alt-m`) — see the leader reference below.

== Leader key — Emacs / Meow keypad

The leader (`<Space>`, also `Alt-m`) mirrors how Meow's *keypad* reaches the
Emacs keymap in `init.el`. Press the same keys after `<Space>` that you would
press after `SPC` in Emacs and you get the IntelliJ action matching that Emacs
command — so `SPC x f` = `C-x C-f` = `find-file`, `SPC c f` = `C-c f` =
`find-file`, and `SPC m x` = `M-x`. Anything without a dedicated key is one
`M-x` away: `SPC m x` opens *Find Action*, exactly as `M-x` runs any command in
Emacs. The which-key popup lists each group as you type.

[cols="1,3",options="header"]
|===
| Prefix | Mirrors

| `SPC x`   | the Emacs `C-x` prefix — files, buffers, windows
| `SPC x p` | `C-x p` — `project.el`
| `SPC c`   | the Emacs `C-c` prefix (`= M-m`) — the personal commands in `init.el`
| `SPC c j` | `C-c j` — avy jumps
| `SPC m`   | the `M-` (Meta) prefix
| `SPC m s` | `M-s` — the consult search family
| `SPC w`   | the window map (Emacs `C-c w`, `SPC w` in Meow)
|===

=== Direct keys & extras

[cols="1,3",options="header"]
|===
| Keys | Action

| `SPC b`    | Buffers (`consult-buffer`) → `Switcher`
| `SPC s`    | Search this file (`consult-line`) → `Find`
| `SPC <CR>` | Context actions (≈ `embark-act`) → `ShowIntentionActions`
| `SPC SPC`  | Alternate file (`<C-^>`)
| `SPC /`    | Toggle line comment
| `SPC 1`–`9` | Numeric count for the next command (`meow-digit-argument`)
|===

Where IntelliJ has no equivalent for an Emacs command the closest action is
used (e.g. `dired` → the Project tool window, `org-agenda` → the TODO tool
window, `org-capture` → a scratch file). `C-x C-c` quits Emacs; here `SPC x c`
closes the project rather than the whole IDE.
